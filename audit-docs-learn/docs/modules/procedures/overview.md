## ðŸ§© Core Concepts

### Procedure vs. Workpaper

- **Procedure**: The plan + execution container for a testing area (e.g., Cash, Revenue, AR Confirmations).
- **Workpaper**: The supporting documentation/evidence and analysis that substantiates the procedureâ€™s conclusion.

In practice, procedures often contain multiple attached or linked workpapers (spreadsheets, PDFs, memos, extracts).

### Assertions and Risk Linkage

Even when the UI simplifies it, your documentation needs to explain:
- What assertion(s) the procedure addresses (existence, completeness, valuation, etc.)
- Which risks the procedure responds to (including significant risks)
- Why the nature/timing/extent is appropriate
